# كيفية نشر هذا المشروع على Vercel

اتبع هذه الخطوات لنشر تطبيقك بنجاح على منصة Vercel.

## المتطلبات الأساسية

1.  **حساب Vercel**: إذا لم يكن لديك حساب، يمكنك التسجيل مجانًا على [Vercel.com](https://vercel.com/).
2.  **حساب Git**: ستحتاج إلى حساب على [GitHub](https://github.com/) أو [GitLab](https://gitlab.com/) أو [Bitbucket](https://bitbucket.org/). (GitHub هو الأكثر شيوعًا).
3.  **تثبيت Git**: تأكد من أن Git مثبت على جهازك المحلي.

---

## الخطوة 1: دفع الكود إلى مستودع Git

1.  **إنشاء مستودع جديد**: اذهب إلى GitHub (أو منصتك المفضلة) وأنشئ مستودعًا جديدًا (New Repository). لا تقم بإضافة `README.md` أو `.gitignore` أو ترخيص من واجهة GitHub.
2.  **ربط المشروع بالمستودع**: في الطرفية (Terminal) داخل مجلد مشروعك، قم بتنفيذ الأوامر التالية. استبدل `YOUR_REPOSITORY_URL` بالرابط الذي حصلت عليه من GitHub.

    ```bash
    git init -b main
    git add .
    git commit -m "Initial commit"
    git remote add origin YOUR_REPOSITORY_URL
    git push -u origin main
    ```

---

## الخطوة 2: الحصول على مفتاح حساب خدمة Firebase (Service Account)

لكي يتمكن خادم Vercel من الوصول إلى قاعدة بيانات Firestore بأمان، ستحتاج إلى إنشاء "مفتاح حساب خدمة".

1.  اذهب إلى [وحدة تحكم Firebase](https://console.firebase.google.com/).
2.  اختر مشروعك.
3.  اضغط على أيقونة الترس (⚙️) بجوار "Project Overview" واختر **Project settings**.
4.  اذهب إلى تبويب **Service accounts**.
5.  اضغط على زر **Generate new private key**.
6.  سيتم تنزيل ملف JSON. **احتفظ بهذا الملف بأمان، فهو يحتوي على صلاحيات الوصول الكاملة لمشروعك.** لا تشاركه أبدًا ولا تضعه في مستودع Git العام.

---

## الخطوة 3: نشر المشروع على Vercel

1.  **إنشاء مشروع جديد**: اذهب إلى لوحة تحكم Vercel الخاصة بك.
2.  اضغط على **Add New...** واختر **Project**.
3.  **استيراد المستودع**: ابحث عن المستودع الذي أنشأته في الخطوة 1 وقم باستيراده (Import).
4.  **إعداد متغيرات البيئة**:
    *   قبل الضغط على "Deploy"، افتح قسم **Environment Variables**.
    *   أضف متغيرًا جديدًا:
        *   **Name**: `FIREBASE_SERVICE_ACCOUNT`
        *   **Value**: افتح ملف JSON الذي قمت بتنزيله في الخطوة 2، انسخ **كامل محتواه** والصقه هنا. يجب أن يبدأ بـ `{` وينتهي بـ `}`.
    *   (اختياري) إذا كنت تستخدم اسم مستخدم وكلمة مرور للمدير، أضفهما هنا أيضًا:
        *   **Name**: `ADMIN_USERNAME`, **Value**: (اسم المستخدم الخاص بك)
        *   **Name**: `ADMIN_PASSWORD`, **Value**: (كلمة المرور الخاصة بك)
5.  **النشر**: الآن، اضغط على زر **Deploy**.

ستقوم Vercel ببناء ونشر تطبيقك. بعد بضع دقائق، سيصبح موقعك متاحًا على رابط عام.

---

## الخطوة 4: (اختياري) التشغيل محليًا

إذا كنت ترغب في تشغيل الخادم محليًا مع اتصال كامل بقاعدة البيانات (لعرض البيانات في الصفحة الرئيسية والبحث)، ستحتاج إلى إعداد متغيرات البيئة على جهازك.

1.  **إنشاء ملف `.env.local`**: في المجلد الرئيسي لمشروعك، قم بإنشاء ملف جديد باسم `.env.local`.
2.  **إضافة متغير البيئة**: افتح ملف `.env.local` وأضف السطر التالي. استبدل `...` **بمحتوى ملف JSON الكامل** الذي قمت بتنزيله من Firebase.

    ```
    FIREBASE_SERVICE_ACCOUNT='{...}'
    ```

    **ملاحظة هامة**: تأكد من وضع محتوى JSON بالكامل داخل علامتي اقتباس فردية (`'`).

3.  **إعادة تشغيل الخادم**: أوقف خادم التطوير (إذا كان يعمل) وأعد تشغيله بالأمر `npm run dev`.

الآن، يجب أن يتمكن التطبيق من جلب البيانات من Firestore وعرضها بشكل صحيح أثناء التطوير المحلي.
